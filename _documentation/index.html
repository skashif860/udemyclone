Installation Last updated: 2019-10-01

ArcInspire is designed to run on both Shared hosting (eg GoDaddy, InmotionHosting, etc) and Virtual Private Server (VPS) (eg DigitalOcean) or Dedicated Server, and is designed to run on Linux Operating Systems.

However, to make optimal use of the software and to be able to customize your resources (eg to be able to install and use the optional FFMpeg for video encoding, we recommend that you use a VPS or Dedicated Server for any reasonable production).

Server Requirements

You need to make sure your server meets the following requirements for the basic setup:

  • PHP >= 7.2.0
  • BCMath PHP Extension
  • Ctype PHP Extension
  • JSON PHP Extension
  • Mbstring PHP Extension
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • GD Extension
  • cURL Extension
  • ZIP Extension
  • Tokenizer PHP Extension
  • XML PHP Extension
  • Apache mod_rewrite
  • memory_limit >= 100MB in PHP.ini
  • post_max_size >= 30MB in PHP.ini
  • upload_max_filesize >= 30MB in PHP.ini
  • file_uploads enabled in PHP.ini
  • MySQL Database

Start Wizard

ArcInspire comes with an installation wizard which, hopefully makes it easy to install. To get started, do the following:

  1. Download the package from your Envato Account
  2. Unzip the package to any convinient location on your local computer
  3. Inside the unzipped package, there is a zip file called upload_version_xx_xx.zip Upload the contents of this ZIP file into your website directory. You can use FileZilla to upload the ZIP file and unzip it in your server, or use any other upload means suitable to you. Just make sure you upload all the contents of this file.
  4. After uploading the files, go to your web browser and access your website url. This will automatically take you to the installation wizard.
  5. Follow the following steps to install the package.

License Information

To install ArcInspire, you need to provide your Envato Username and Purchase Code. Follow these instructions to obtain the Purchase Code for this item.

Enter the Purchase Code and Username in the form on the START screen and click on "Submit" to validate the license information. If the verification is correct, you can click the "Next" button to proceed with the installation.

Server Requirements

ArcInspire is developed with Laravel v6.0 (The latest version). Therefore to get the script working, your server must meet these basic requirements for running a laravel application.

Besides the basic server requirements for running Laravel, there are other requirements that your server must meet, such as values from PHP.ini file to handle file uploads, folder permissions etc. On the "System Requirements" screen, you can see whether or not your server meets these requirements. Please ensure that you see a "Green Check mark" against each requirement before you proceed.

Database Connection

The script requires MySQL database. On your server, create a new MySQL database, copy the hostname, username, password and port of your database connection, and enter them in the "Database Configuration" form as shown below:

Then click on "Connect to Database" to establish the connection. If the connection is successful, you can click on "Next" to proceed. Otherwise, review the information entered and try again.

Admin User

In the "SETTINGS" stage, you can create the Administrator User Account and also provide the site name. Not the credentials of the Admin User as you will need them to login to the Admin panel.

Mail Settings

You must provide working MAIL settings to enable the script send system emails. If your mail settings are wrong, users will not be able to register to the site (for example), as they need to receive a confirmation email to verify their account when they register.

ArcInspire can work with any of the default Laravel mail drivers - SMTP, Mailgun, Sparkpost, Postmark, SES and Sendmail. On the form, select the mail driver you would like to use and enter the relevant information for the selected driver.

NB: The free version of Mailgun should be sufficient for this script as it provides 10,000 free emails / month.

If you are using SMTP, you can use a service like SMTPer.net to test your credentials.

Finish

On the FINISH screen, you can choose to Import Sample Data to your site. This will add basic site data (similar to the demo site) to your database to get you started. You will be able to relete all the sample data from the Admin Panel with just a single click.

Then just click "Finish" to complete your installation. Once done, you will be redirected to the homepage.

Site Configuration

Once ArcInspire is installed, it is advisable to login as Admin and setup the configuration for the entire site. There are default configuration values entered during installation, but these are minimal.

  1. Login to the site with the admin user you created during installation. You will be redirected to the Admin Dashboard.
  2. In the Admin Dashboard, click on "Settings" on the left side menu.
  3. Then go through the various setting options as described below

Site Settings

Site Info

Under Site Settings, you need to enter the following:

  • Site Mode: If checked, the site will be set to demo mode and this will disable certain requests (eg all DELETE, STORE and UPDATE operations will be disabled).
  • Redirect to HTTPS: If checked, all site assets will be served via HTTPS. Ensure that you have an SSL Certificate installed on your server else links will be broken.
  • Site URL: It is important that you enter the correct site URL (your site domain name). This will be used in all email links (eg for account activation). The default is the current site domain.
  • Site Name: This is the site identifier and will appear on the Browser Title, Footer, system emails etc.
  • Site Description and Keywords: These are used in the Description and Keywords Meta Tags of your site.
  • Site Currency: This is the default currency of your site. All course purchases will be charged using this currency. All other currencies will only be used to display conversions to the user, but ALL charges will be in the Site Currency. You can change the dropdown list by adding or removing currencies under the "Currency Settings" page.
Author Earnings

This specifies the percentage value of each sale that should go to the course instructor.

  • Organic Sales Percen: Percentage paid to course instructor when the sale occurs organically (ie when the sale does not result from the Instructor's PROMO code.)
  • Promo Sales Percent: Percentage paid to course instructor when the course is purchased using a discount coupon created by the instructor.
Images

Here, you can upload the site Logo, Favicon and the banner image that appears on the homepage of the site.

  • Logo: Should have a transparent background.
  • Favicon: Square image, .png or .ico
  • Homepage Banner Image:, preferably 1440x512 in size.

Video Content Settings

Queues

Here, you can manage how you want the application to process jobs. This is especially how you want video uploads to be handled. There are two options:

  • Database: With this option, when a user uploads a video, it will be logged in a queue in the database and processed accordingly. The user will not have to wait for the video upload to complete. The video will be placed in a queue and processed in the background. We highly recommend that you choose this option so as not to keep users waiting for video uploads to complete (especially if you are using FFMpeg to process videos).
    NB If you decide to use Database Driver for queuing (and we recommend that you do), then in your server, you will need to set up a cron job with the following expression:
    */5 * * * * /path-to-php/php /path-to-your-website/artisan schedule:run >> /dev/null 2>&1

    For instance:

    */5 * * * * /usr/local/bin/php /home/myusername/public_html/artisan schedule:run >> /dev/null 2>&1

    This command will execute the task scheduler every 5 minutes and process any videos that are in the queue. The Scheduler runs an internal command to process videos every 10 minutes.

  • Sync: With the Sync queue processing, nothing is queued for later. All queues are processed instantly and the user will need to wait for the process to complete before they proceed with the video upload process.
Video Settings
  • Video Provider: With ArcInspire, Intructors can Upload video content to their courses but they can also use YouTube videos. By default, both Video Upload and YouTube are enabled. If you would want to serve only one of these options, select the corresponding choice.
  • Enable video encoding: This option is specifically for anyone running their site on a server where they have control to install FFMpeg for video processing. FFMpeg is a PHP package that allows for video processing (encoding). If you select this option, then each video uploaded by the users will be processed in 2 different definitions - a large and a small sized video. This is good if you wish to serve videos in both HD and SD versions. If Video Encoding is enabled, then you can also specify the HD and SD dimensions for videos (defaults as 1280x720 for 720HD videos and 640x360 for 360SD videos).
  • Video Max Size: This is the maximum size of video that an Instructor can upload. The default is 20MB. NOTE: Ensure that the post_max_size and file_upload_size parameters in your PHP.ini are greater than the value you choose here to allow your server handle large file uploads.
  • Video Upload Location: You can choose to save uploaded videos to your local server or to Amazon s3 bucket. If you choose to upload to Amazon s3, then you will also need to provide the Amazon s3 credentials such as AWS Access ID, AWS Region, s3 Bucket and AWS URL (optional)

Site Currency Settings

ArcInspire supports multiple currencies in the frontend, but all payment is only collected in the main currency. Users will be able to switch currencies and view prices in other currencies base on the settings in the admin panel.

Each currency has a Conversion Rate. The Conversion Rate of the Default Currency must be equal to 1. The conversion rates of all other currencies should be relative to the default currency.

Payment Gateways

At the moment, ArcInspire supports PayPal payment and Credit Card Processing. You can choose to use one or both ot these options.

PayPal Payment

In order to use PayPal payment, you need to create and provide your PayPal API client ID and Secret Key. You need to enter both the Demo (Sandbox) keys as well as the LIVE keys. Follow these steps to obtain the API credentials on PayPal:

  1. Go to PayPal Developer Account and login
  2. In the "My Application & Credentials" menu click on "Create App" button to create a new all.
  3. Under the "App Settings", make sure that the the following features are enabled:
    • Accept Payments
    • Invoicing
    • Payouts
    • Disputed
  4. Copy the Client ID and Secret keys and enter them in your Admin Dashboard.
  5. Do same for both LIVE and SANDBOX keys. Note that for LIVE keys, you may have to manually request PayPal to add the Payouts feature.
Credit Card Processors

At the moment, there are two credit card processors available - Stripe and RazorPay. You can only choose to use on of these options, and in each case, you also need to provide the corresponding API Keys.

Stripe

For Stripe, you can go to Stripe.com to create a Free account if you do not have one yet. Once an account is created, you can follow the documentation on the Stripe Help page to obtain the API Credentials for both Sandbox (Test) and Live environments.

RazorPay

This is a Payment Gateway for India and supports multiple currencies. You can go to Razorpay.com to create a Free account if you do not have one yet. Once an account is created, you can follow the documentation on the RazorPay Help page to obtain the API Credentials for both Sandbox and Live environments.

Note: Make sure to turn your Payment Modes (PayPal mode, Stripe Mode and/or RazorPay mode) from "Sandbox" to "Live" when you are ready to start collecting live payments.

Financial Transactions

ArcInspire offers the possibility for course purchase, management of discount coupons, processing of purchase refunds and paying out earnings to course instructors.

Discount Coupons

There are two categories of discount coupons - Sitewide Coupons and Individual Course Coupons

Sitewide Coupons

This is a coupon that is created by the site Administrator and can be applied to any course on the platform. When a user goes to their shopping cart, the system checks if there is any valid sitewide coupon in the database and if there is, it automatically applies the coupon to the shopping cart. Sitewide discount coupons can be managed from the Admin panel via the Discount Coupons menu link. There can only be one active Sitewide coupon at any given time.

If a course is purchased using a Sitewide coupon, then the sale is considered to be Organic, in which case the instructor's commission from that sale is calculated based on the Organic sales percent in the Author earnings setting of the admin panel (default is 40%).

Course-specific Coupons

These coupons are created by the course intructor in their course management dashboard, and can be applied only to the specific course. The instructor can create as many coupons for a course as they want.

If a course is purchased using a course-specific coupon, then the sale is considered to be Promo, in which case the instructor's commission from that sale is calculated based on the Promo sales percent in the Author earnings setting of the admin panel (because it means the sale originated from the instructor's promotion).

When admin or intructors want to create a new coupon, they are presented with two options - Either a "FREE" coupon (which students can use to enroll to the course for free), or a "Discounted Price" which is a percentage of the price that should be reduced (minimum is 5%).

Purchasing a Course

ArcInspire comes with a shopping cart. Multiple courses can be purchased at once in a cart, and one or more discount coupons can be applied to the cart. If a course price is 0 (Free), then the user does not go through the cart but is directly presented with a button to enroll into the course. Otherwise, the user can proceed to the shopping cart (where they can add/remove coupons or remove items from the cart) and then proceed to the checkout page.

Course purchase can be done with PayPal or Credit Card. Please see the Payment Gateways section for how to setup the payment providers in the admin panel.

Refunds and Periods

Instructors are paid monthly, following pre-generated 30-day periods. Once a user makes a purchase on the site, each payment is attached to a specific period - usually 2 months ahead of the current period. This is because the systems allows a 30-day money-back guarantee for all purchases. So once a purchase is made, the student has 30 days to request a refund (no questions asked). So if a user requests a refund, the administrator can process it directly in the admin panel. Once processed, the student is unenrolled to the course and any reviews the student might have written for the course are deleted. The student is also notified by email.

All refunds are made directly to the same payment method the user used to make the initial purchase. So if a course was purchased with PayPal, the refund will be made directly to the same PayPal account. Same with credit cards.

Once a payment period is over, the Admin will be able to "Close the period" from the admin panel under the Finance > Payouts menu. You will not be allowed to close a period that has not ended. Once the period is closed, all payouts for all users are calculated for that period immediately (taking into account any refunds that might have been made for that period)

Instructor Payouts

Once a payment period is closed and payouts automatically calculated, the Admin will see a list of all payouts for the site. Any payout with the value of 0 is automatically marked as "Completed". ALL PAYOUTS ARE MADE USING PAYPAL. So if an instructor has a payout but has not provided their PayPal Email Accounts (which they can do in the user account settings in the frontend), the Admin will be presented with a button to send an Email reminder to the Instructor to enter their PayPal email address.

If an instructor has a Payout greater than 0 and has provided their PayPal account, they admin will be able to directly process the PayPal payment from ArcInspire.

NOTE: Before processing any payouts, ensure that you have enough money in the Admin PayPal account to send out (Of Course!).

Once a payment is sent to the instructor, the status is marked as "Complete" and the instructor is notified by email.

Updates

ArcInspire provides an easy way for the Admin to update the application

Updating the application

  • Go to the Admin Panel. Once in the admin panel, the application will automatically check for updates on our remove server.
  • If an update exists, you will see a notice with a button: Update Now!. Click on this button and wait for the update to complete.
